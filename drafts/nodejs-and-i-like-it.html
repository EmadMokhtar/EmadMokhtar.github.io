Nodejs and I like it

Posted on Mon 31 December 2018 in python, nodejs

I had a chance to try Node.js framework last month; I decided to build REST APIs using Node.js and express.js. Regularly I'm using JavaScript in my daily work. I'm using Django to build Web Applications, but many times I need to develop a front-end feature, I need to use JavaScript. I'm always using JavaScript to write front-end logic, but I never wrote back-end logic using it. I took the challenge to develop a simple REST APIs using express.js.

===Challenge accepted image==

Development environment

To start developing, I need to get my machine ready, so I installed node.js via Homebrew (I'm a macOS user), it is quite easy, and node.js installation comes with npm "npm is a package manager for node.js it is like pypi or pip for Python". I used Visual Studio Code as a text editor.

What I've learned?

I learned many things, especially that I'm experiencing the new dev stack.

Node.js

Node is a backend engine; it is Google Chrome V8 engine running a JavaScript on your machine instead of a browser. Node is not a language it is an engine.

Expressjs

It is a web framework that runs via Node.js; it is a framework that receives requests and replies with responses. It is Micro-framework, you can say it is like Flask, micro-framework means it doesn't have ORM, Template engine, etc.

ORM

I tried to build my models using sequelize JS as I'm more familiar with PostgreSQL DB engine, but it turns to be so tricky to implement my feature using RDBMS system. I switch to using Mongoose JS and my DB engine is MongoDB. I was wrong from the beginning, but after switching to MongoDB, it is much easier to implement the feature, and MongoDB is a preferred DB in Node.js community.

Asynchronous Programming

Get the used of the nature of JavaScript asynchronous "Promises" was a big challenge for me, it is a complete paradigm shift. On daily basis I'm using Django, and it is not asynchronous by default "there is a roadmap for Django to be Async". Python has async programming since 3.5, but frameworks need to adopte it to become async as well, like requests-future.

Final Thoughts

I enjoyed using Node.js; the community is excellent, there is nothing you can't find a package for it on NPM, it is such an active community, but what I don't like in the community is the quality of tutorial, it is such naive tutorials, yes we need naive tutorial, but what about the advanced tutorial.

The async programming challenge was an eye-opener for me; I need to dig deep in this way of programming especially in Python. I'm lucky that there is a video course on Talk Python about asynchronous in Python, and I think I will watch it.
